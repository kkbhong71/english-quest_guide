바이브 코딩(Vibe Coding)이란?

AI에게 자연어로 원하는 것을 설명하면, AI가 코드를 생성해주는 새로운 개발 방식입니다.

💡 핵심 개념

바이브 코딩은 "내가 코드를 작성하는 것"이 아니라 "AI에게 무엇을 만들어 달라고 요청하는 것"입니다. 프로그래밍 문법을 몰라도, 만들고 싶은 것을 명확하게 설명할 수 있다면 웹앱을 완성할 수 있습니다.

전통 코딩 vs 바이브 코딩

구분전통 코딩바이브 코딩
필요 지식프로그래밍 언어 문법만들고 싶은 것을 설명하는 능력
도구VSCode, 터미널Claude, ChatGPT 등 AI
디버깅직접 코드 분석오류 메시지를 AI에게 전달
소요 시간수일~수주수시간~1~2일
핵심 역량코딩 실력기획력 + 프롬프트 작성력

좋은 프롬프트 작성법

📌 프롬프트의 4가지 요소

① 목적: 무엇을 만들 것인가 (영어 교육용 게임)
② 대상: 누가 사용하는가 (중학교 기초 부족 학생)
③ 기능: 어떤 기능이 필요한가 (발음, 따라쓰기, 퀴즈)
④ 제약: 기술적 조건 (단일 HTML, GitHub Pages 배포)

Step 1: 개발 환경 준비

GitHub 계정과 저장소만 있으면 됩니다. 별도 설치는 필요 없습니다.

1
GitHub 계정 생성
github.com에서 무료 계정을 만듭니다.
2
새 저장소(Repository) 생성
New Repository → 이름: english-quest → Public 선택 → Create
3
index.html 파일 업로드
Add file → Upload files → AI가 생성한 index.html을 드래그하여 업로드
4
GitHub Pages 활성화
Settings → Pages → Source: Deploy from a branch → Branch: main → Save
⚠️ 주의사항

GitHub Pages는 Public 저장소에서만 무료입니다. Private 저장소는 GitHub Pro 플랜이 필요합니다. 배포까지 1~2분 정도 걸립니다.

❌ Babel 브라우저 변환의 함정

초기에 <script type="text/babel">로 작성하면, 브라우저가 JSX의 < 비교 연산자를 HTML 태그로 오인하여 화면 하단에 코드가 노출되는 치명적 버그가 발생합니다. 반드시 Babel CLI로 사전 컴파일하여 일반 JavaScript로 변환해야 합니다.

📌 데이터 저장 방식: localStorage

XP와 레벨 시스템

JavaScript - 레벨 시스템
// 레벨별 필요 XP 공식
var xpFor = function(lv) {
  return lv * 50 + (lv - 1) * 20;
};

// Lv.1 → 50 XP, Lv.2 → 70 XP, Lv.3 → 90 XP ...
// 점점 많은 XP가 필요해서 학습 동기 유지

// XP 획득 및 레벨업 체크
var addXP = function(amount) {
  var newXP = student.xp + amount;
  var newLevel = student.lv;
  while (newXP >= xpFor(newLevel + 1)) newLevel++;
  // 레벨업 시 축하 메시지 표시
};

🔊 Web Speech API (TTS) — 모바일 대응

JavaScript - 모바일 TTS 해결
// 문제: 모바일에서 소리가 안 나는 이유
// → iOS/Android 보안 정책: 사용자 터치 없이 음성 재생 차단

// 해결 1: 음성 목록 사전 로딩
var ttsVoice = null;
function initTTS() {
  var loadVoices = function() {
    var voices = speechSynthesis.getVoices();
    ttsVoice = voices.find(function(v) {
      return v.lang === "en-US";
    });
  };
  loadVoices();
  speechSynthesis.onvoiceschanged = loadVoices;
}

// 해결 2: 첫 터치 시 TTS 잠금 해제
document.addEventListener('touchstart', function() {
  var u = new SpeechSynthesisUtterance("");
  u.volume = 0;  // 무음
  speechSynthesis.speak(u);  // 잠금 해제!
}, { once: true });

// 해결 3: cancel과 speak 사이 딜레이
function speak(text) {
  speechSynthesis.cancel();
  setTimeout(function() {
    var u = new SpeechSynthesisUtterance(text);
    u.lang = "en-US";
    u.rate = 0.85;  // 천천히
    if (ttsVoice) u.voice = ttsVoice;
    speechSynthesis.speak(u);
  }, 50);  // 50ms 딜레이
}

HTML - 비밀번호 분리 패턴

<script>
  window.TEACHER_PW = "5012";  // ← 여기만 변경!
</script>

<!-- 앱 코드에서는 이렇게 참조 -->
<script>
  if (inputPW === (window.TEACHER_PW || "5012")) {
    // 로그인 성공
  }
</script>
💡 왜 비밀번호를 분리하는가?

Babel로 사전 컴파일된 코드는 수천 줄이어서 비밀번호 위치를 찾기 어렵습니다. window.TEACHER_PW를 HTML 상단에 분리하면 메모장으로도 쉽게 변경할 수 있습니다. 이 패턴은 English Quest 이전 프로젝트인 "영어 퀘스트"에서도 동일하게 적용되었습니다.

Step 8: 모바일 최적화

학생들은 대부분 스마트폰으로 접속합니다. 모바일 경험이 가장 중요합니다.

체크리스트

  • viewport 메타태그로 확대/축소 방지
  • position: sticky 헤더로 네비게이션 항상 노출
  • touch-action: none으로 캔버스 터치 시 스크롤 방지
  • touchstart 이벤트에서 TTS 잠금 해제
  • 최소 터치 영역 44px 이상 유지
  • iOS Safari -webkit-backdrop-filter 대응
  • mobile-web-app-capable 메타태그 (deprecated 메타태그 교체 주의)

트러블슈팅 — 실제 겪은 문제들

English Quest 개발 과정에서 실제로 발생한 버그와 해결 방법입니다.

🐛 1. 화면 하단에 코드가 노출됨

원인: <script type="text/babel"> 사용 시, 브라우저가 JSX의 <를 HTML 태그로 오인

해결: Babel CLI로 사전 컴파일 → <script> (type 속성 없이) 사용

🐛 2. 모바일에서 발음이 안 나옴

원인: iOS/Android 보안 정책 — 사용자 제스처 없이 음성 재생 차단

해결: ① 첫 터치 시 무음 TTS로 잠금 해제 ② onvoiceschanged로 음성 목록 비동기 로딩 ③ cancel()과 speak() 사이 50ms 딜레이

🐛 3. 교사 비밀번호 변경이 안 됨

원인: 비밀번호가 컴파일된 코드 깊숙이 하드코딩되어 수정 위치를 찾기 어려움

해결: window.TEACHER_PW를 HTML 상단의 별도 <script> 블록에 분리

🐛 4. favicon 404 오류

원인: 기본 /favicon.ico 파일이 없음

해결: <link rel="icon" href="data:image/svg+xml,...">로 인라인 SVG 아이콘 설정

🐛 5. apple-mobile-web-app-capable deprecated 경고

원인: apple-mobile-web-app-capable 메타태그가 더 이상 사용되지 않음

해결: mobile-web-app-capable로 변경
